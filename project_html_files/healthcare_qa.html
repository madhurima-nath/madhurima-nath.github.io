Healthcare Q & A virtual assistant

A healthcare technology start-up client wanted to improve their application interface by incorporating Generative AI to boost customer experience. At the current state, the interface performed clustering on the bio-pharmaceutical and medical technical datasets and generated plots representing the different groups. A Q & A chatbot using Anthropic Claude LLM (Large Language Model) was developed on AWS Bedrock to assist both experts and non-medical users to easily gain insights from these plots.
Ingesting data as-is in the LLM prompt increased latency, token usage and thus cost. Pre-processing data to extract relevant information and summarising it helped to improve accuracy, optimise for latency, and return an answer in ~20 seconds on average. Implementing LLM agent with carefully designed prompt and the appropriate tools provided a flexible framework to answer various types questions, as well as the capability to expand the scope of questions in the future.
Note: Collaborated with design team, who developed the front-end interface.

  • The numeric results from the clustering analysis were parsed to extract the relevant information from the structured json heatmaps.
  • LLM pipeline #1: LLM chain to summarize descriptions of characteristics of each group which represent various medical conditions or symptoms.
  • LLM pipeline #2: LLM chain to propose a name for each group/clusters from the graphs.
  • LLM pipeline #3: LangChain agent to answer questions about the groups.
  • A custom agent to interact with AWS Bedrock and Anthropic Claude.
  • Graph specific question tools to gain answers from the heatmaps.
    • At the end of the engagement, a fully functioning chatbot was delivered that could be integrated with their existing application interface.
