Data Engineering on Databricks

An end-to-end financial data pipeline on Databricks, built in three layers: raw ingestion, cleaning and joining, and analysis-ready aggregates. It takes in US equity and macroeconomic data — S&P 500 daily prices, Federal Funds Rate, and the VIX volatility index — to examine how equity markets respond to changes in monetary policy. GitHub Copilot and Databricks Assistant were used throughout: pipeline architecture and join logic were designed by hand; code was written with AI tools.

The project is built on Databricks Community Edition, with the code structured to be Unity Catalog ready — minimal changes required to run on a premium workspace.

Medallion Architecture

Data flows through three layers, each with a distinct responsibility:

  • Bronze. Raw ingestion from source files — S&P 500 OHLCV from Yahoo Finance, monthly Federal Funds Rate from FRED, and VIX from CBOE. No transformations; source fidelity is preserved.
  • Silver. Cleaning, type casting, and the core join logic. The main technical challenge here is aligning data of different frequencies: the Fed Rate is published monthly, the S&P 500 is daily. A point-in-time join using a SQL window function carries the most recent known rate forward into every daily row.
  • Gold. Strategy-ready aggregates: 30-day rolling volatility of S&P 500 returns, correlation between daily returns and the current rate, and rate regime classification (rising, falling, or stable).

Point-in-Time Join

Joining a monthly time series (Fed Rate) into a daily one (S&P 500) requires carrying the most recent known value forward — standard equi-joins produce nulls on most rows. The Silver layer solves this with a SQL window function:

Forward-fill — Silver layer 
WITH combined AS (
  SELECT p.date, p.close AS sp500_close, r.fed_rate
  FROM bronze.sp500_prices p
  LEFT JOIN bronze.fed_rates r ON p.date = r.date
),
filled AS (
  SELECT date, sp500_close,
    LAST_VALUE(fed_rate, true) OVER (
      ORDER BY date
      ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS current_fed_rate
  FROM combined
)
SELECT *,
  (sp500_close - LAG(sp500_close) OVER (ORDER BY date))
    / LAG(sp500_close) OVER (ORDER BY date) AS daily_return
FROM filled;

The result is a Silver table where every daily S&P 500 row carries the rate in effect on that day, and the daily percentage return is computed in the same pass.

AI-Augmented Development

Two AI tools were used throughout:

  • GitHub Copilot generated the PySpark transformation scripts for each layer, given a description of the input schema and the required output. The forward-fill join logic, schema validation functions, and Gold aggregation queries were all produced this way.
  • Databricks Assistant was used for SQL optimisation and to scaffold the Streamlit dashboard — iterating on chart layout and connection logic against the Unity Catalog schema.

The pipeline architecture, join strategy, and data quality rules were specified by hand; code generation was delegated to the AI tools. This keeps the reasoning behind each design decision explicit and easy to verify.

Dashboard

A Streamlit dashboard connects to the Gold layer to visualise the relationship between US monetary policy and equity market behaviour. Key views:

  • S&P 500 price and Federal Funds Rate plotted on a shared time axis — showing the lag between rate decisions and market response.
  • VIX overlaid with rate change events — visible spikes in volatility around the 2022 hiking cycle.
  • 30-day rolling volatility as a function of rate regime (rising, falling, stable).
