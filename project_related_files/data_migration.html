Financial Analytics Pipeline on Databricks

Financial institutions have been running market analytics in SAS for decades. Moving to a modern data platform requires two things: a pipeline to replace the old infrastructure, and a tool to migrate the legacy code. This project builds both, using 15 years of real US and European market data.

The pipeline ingests five data series — S&P 500, Euro Stoxx 50, VIX (a measure of market uncertainty), US Federal Funds Rate, and ECB Deposit Facility Rate — processes them through a three-layer Databricks medallion architecture (Bronze, Silver, Gold), and serves the results through an interactive dashboard. A companion tool takes legacy SAS analytics code and converts it to PySpark or Databricks SQL automatically.

Medallion Architecture

Data flows through three Delta Lake layers, each with a distinct role:

  • Bronze. Raw ingestion from two sources: Yahoo Finance (daily price and volume data for S&P 500, Euro Stoxx 50, and VIX) and FRED (Federal Reserve Economic Data — monthly US Federal Funds Rate and ECB Deposit Facility Rate). Five Delta tables written without any transformation — source data is preserved exactly as received.
  • Silver. Quality checks, cleaning, and a mixed-frequency join that aligns monthly central bank rates onto the daily equity date spine. Log returns and the US–EU rate differential are computed here. A reusable quality check module runs at every layer transition and halts the pipeline on any failure.
  • Gold. Analysis-ready aggregates: rolling realised volatility (20-day and 60-day, annualised) for both indices, 60-day rolling correlation between US and EU equity returns, S&P 500 drawdown from its 52-week high, VIX stress regime, US and EU rate regimes, and a policy divergence classification.

Mixed-Frequency Join

The Fed Funds Rate and ECB rate are published monthly; equity data is daily. A standard date join leaves most rows null because the rate observation dates rarely fall on trading days. The Silver layer resolves this with a two-pass window function: first a forward-fill to carry each rate reading forward through subsequent trading days, then a backward-fill to handle the initial days at the start of the series before the first rate observation date.

Forward- and backward-fill — Silver layer (PySpark) 
fill_window = Window.orderBy("date").rowsBetween(
    Window.unboundedPreceding, Window.currentRow
)
back_window = Window.orderBy("date").rowsBetween(
    Window.currentRow, Window.unboundedFollowing
)

joined = daily_df.join(monthly_df, on="date", how="left")
filled = (
    joined
    .withColumn("fed_rate",
        F.last(F.col("fed_rate"), ignorenulls=True).over(fill_window))
    .withColumn("fed_rate",
        F.first(F.col("fed_rate"), ignorenulls=True).over(back_window))
)

Each trading day carries the rate in effect on that day — a point-in-time join with no lookahead bias.

Gold Layer

The Gold layer computes all metrics needed by the dashboard in a single PySpark pass over the Silver table. Rolling statistics use explicit window sizes (20 and 60 trading days) rather than calendar periods, which gives consistent behaviour across holiday gaps. Regime classifications use threshold rules calibrated to the 2010–present rate environment — the ECB rate turned negative in 2014, so the EU regime labels include a “negative” band not present in the US classification.

SAS → PySpark Converter

A standalone tool that takes legacy SAS code and outputs PySpark DataFrame API, Databricks SQL, or dbt YAML. Common SAS patterns — PROC SORT, PROC MEANS, PROC SQL, DATA steps with KEEP, DROP, RENAME, WHERE, IF-THEN-ELSE, and date functions — are handled by a deterministic rule engine. Code that falls outside the rule set is passed to an AI model via the Anthropic API, which returns a conversion with a review note flagging anything that needs human verification.

The converter is tested with 23 pytest cases covering every supported pattern. It is accessible from the dashboard without any local setup.

Dashboard

A Streamlit app deployed at financial-analytics-databricks.streamlit.app. It connects directly to the Gold Delta table on Databricks — no local data files needed.

  • Analytics Dashboard. The output of the pipeline — interactive charts covering equity indices, central bank rates, realised volatility, US–EU equity correlation, S&P 500 drawdown, and current regime classifications across 15 years of daily data. Date range is filterable.
  • SAS → PySpark Converter. The migration tool — paste legacy SAS code, select the target format (PySpark, Databricks SQL, or dbt YAML), and get the converted output with notes and any warnings.
