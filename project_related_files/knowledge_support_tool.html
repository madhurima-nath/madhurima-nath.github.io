GenAI & LLM Applications

A retrieval-augmented generation (RAG) pipeline was deployed across three client engagements in financial services, equipment rental, and healthcare. The core architecture — document ingestion into a vector store, semantic retrieval, and LLM-generated responses — was adapted to each sector's data types, compliance requirements, and end-user workflows.

Product Discovery — Financial Services

A major financial institution needed a way for internal teams to search and discover relevant products across a large, complex catalogue. Manual search was slow and inconsistent; the goal was a system that could understand natural language queries and surface the most relevant products with supporting context.

The solution was built on Databricks, using Llama as the LLM and LangChain's agent framework to handle multi-step reasoning over retrieved documents. MLflow was used throughout for experiment tracking and pipeline management. The system was built in close coordination with compliance teams to ensure it met the institution's regulatory requirements for information handling. The deployed system delivered measurable improvements in information retrieval accuracy and speed.

Knowledge Support Tool — Equipment Rental

A leading equipment rental firm built a knowledge support tool as their first generative AI initiative, aimed at helping sales representatives respond to customer inquiries faster, reducing manual document searches, and serving as an internal training resource. Subject matter experts (SMEs) also needed the ability to review and correct any response flagged as inaccurate by end users.

The knowledge base was built from the firm's existing documents — both structured (CSV, JSON) and unstructured (PDF, HTML) — ingested into an S3 bucket and embedded using a GPT-3.5 model. Chroma was used as the vector store. The RAG pipeline was implemented using LangChain's RetrievalQA Chain, with a prompt template designed to match the client's requirements: answers formatted as lists where appropriate, with hyperlinks to source documents included in responses.

The deployment ran on AWS. Multiple Lambda functions handled distinct parts of the pipeline — processing input, returning LLM responses, and updating the database with SME feedback. LangChain dependencies were containerised via Docker, deployed through ECR to an EC2 instance, and connected to AWS Lex for NLP and NLU. The SME review interface connected to the backend via API Gateway; metadata and SME feedback were stored in DynamoDB.

The tool was tested extensively by the firm's staff across a wide range of questions and produced accurate results throughout. Customer inquiry response time was reduced by 22%.

Clinical Q&A — Healthcare

A Q&A system was built for a biopharma and medical technology client using Claude on AWS Bedrock to extract insights from clinical and technical datasets. Developed in partnership with the client's design teams to improve interface accessibility and user experience.
